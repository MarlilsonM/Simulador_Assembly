Documentação dos Comandos

Introdução

Bem-vindo à documentação do nosso Simulador de Assembly! Esta documentação foi elaborada para fornecer informações abrangentes sobre os comandos disponíveis no editor de código do projeto, permitindo que você escreva, execute e depure programas em linguagem assembly de forma eficiente.

Você encontrará:

  • Lista de Comandos: Uma descrição detalhada de cada comando disponível, incluindo exemplos de uso e sintaxe.
  • Funcionalidades do Simulador: Informações sobre como utilizar as principais funcionalidades, como o editor de código, depuração e visualização de execução.
  • Guias de Contribuição: Instruções sobre como contribuir para o projeto, sugerir melhorias ou relatar problemas.
  • Exemplos Práticos: Exemplos de código que demonstram como utilizar os comandos e funcionalidades do simulador.

Se você é novo no desenvolvimento em assembly ou no uso deste simulador, recomendamos começar pela seção de exemplos práticos para se familiarizar com a sintaxe e as funcionalidades. Para qualquer dúvida ou feedback, sinta-se à vontade para entrar em contato através do nosso repositório no GitHub.

Comandos Disponíveis

Nome do Comando Descrição Sintaxe Exemplo Notas
MOV Move um valor para um registrador ou da memória. MOV <destino>, <origem> MOV r0, 5 O valor 5 será movido para o registrador r0.
LOAD Carrega um valor da memória para um registrador. LOAD <destino>, [<endereço>] LOAD r0, [10] O valor armazenado no endereço 10 será carregado no registrador r0.
STORE Armazena um valor de um registrador ou um valor direto em um endereço específico da memória. STORE <origem>, [<endereço>] STORE r0, [10] O valor contido em r0 será armazenado no endereço de memória 10. Se a origem não for um registrador, um valor direto será armazenado.
ADD Adiciona dois valores e armazena o resultado no registrador de destino. ADD <destino>, <origem> ADD r0, r1 O valor de r1 será adicionado ao valor de r0 e armazenado em r0.
SUB Subtrai o valor do registrador de origem do registrador de destino. SUB <destino>, <origem> SUB r0, r1 O valor de r1 será subtraído de r0 e o resultado será armazenado em r0.
MUL Multiplica dois valores e armazena o resultado no registrador de destino. MUL <destino>, <origem> MUL r0, r1 O valor de r0 será multiplicado pelo valor de r1 e armazenado em r0.
DIV Divide o valor do registrador de destino pelo valor do registrador de origem. DIV <destino>, <origem> DIV r0, r1 O valor de r0 será dividido pelo valor de r1 e o resultado será armazenado em r0.
AND Realiza a operação lógica AND entre dois registradores. AND <destino>, <origem> AND r0, r1 O resultado da operação AND entre r0 e r1 será armazenado em r0.
OR Realiza a operação lógica OR entre dois registradores. OR <destino>, <origem> OR r0, r1 O resultado da operação OR entre r0 e r1 será armazenado em r0.
XOR Realiza a operação lógica XOR entre dois registradores. XOR <destino>, <origem> XOR r0, r1 O resultado da operação XOR entre r0 e r1 será armazenado em r0.
NOT Inverte os bits do valor em um registrador. NOT <destino> NOT r0 Os bits do valor em r0 serão invertidos.
CMP Compara dois valores e atualiza as flags. CMP <op1>, <op2> CMP r0, r1 Compara os valores de r0 e r1, atualizando as flags de acordo.
JMP Salta para uma etiqueta especificada no código. JMP <etiqueta> JMP inicio O fluxo de execução será transferido para a linha marcada com a etiqueta "inicio".
JE Salta para uma etiqueta se a comparação anterior for igual. JE <etiqueta> JE sucesso Se a última comparação resultou em igualdade, salta para a etiqueta "sucesso".
JNE Salta para uma etiqueta se a comparação anterior não for igual. JNE <etiqueta> JNE erro Se a última comparação não resultou em igualdade, salta para a etiqueta "erro".
JG Salta para uma etiqueta se a comparação anterior for maior. JG <etiqueta> JG maior Se a última comparação indicou que o primeiro valor é maior, salta para a etiqueta "maior".
JL Salta para uma etiqueta se a comparação anterior for menor. JL <etiqueta> JL menor Se a última comparação indicou que o primeiro valor é menor, salta para a etiqueta "menor".
JGE Salta para uma etiqueta se a comparação anterior for maior ou igual. JGE <etiqueta> JGE sucesso Se a última comparação indicou que o primeiro valor é maior ou igual, salta para a etiqueta "sucesso".
JLE Salta para uma etiqueta se a comparação anterior for menor ou igual. JLE <etiqueta> JLE erro Se a última comparação indicou que o primeiro valor é menor ou igual, salta para a etiqueta "erro".
JZ Salta para uma etiqueta se a comparação anterior resultou em zero. JZ <etiqueta> JZ igual Se a última comparação resultou em zero, salta para a etiqueta "igual".
JNZ Salta para uma etiqueta se a comparação anterior não resultou em zero. JNZ <etiqueta> JNZ diferente Se a última comparação não resultou em zero, salta para a etiqueta "diferente".
JC Salta para uma etiqueta se a flag de carry estiver setada. JC <etiqueta> JC carry Se a flag de carry estiver setada, salta para a etiqueta "carry".
JNC Salta para uma etiqueta se a flag de carry não estiver setada. JNC <etiqueta> JNC semCarry Se a flag de carry não estiver setada, salta para a etiqueta "semCarry".
JO Salta para uma etiqueta se a flag de overflow estiver setada. JO <etiqueta> JO overflow Se a flag de overflow estiver setada, salta para a etiqueta "overflow".
JNO Salta para uma etiqueta se a flag de overflow não estiver setada. JNO <etiqueta> JNO semOverflow Se a flag de overflow não estiver setada, salta para a etiqueta "semOverflow".
JB Salta para uma etiqueta se a comparação anterior for menor (usando a flag de carry). JB <etiqueta> JB menor Se a última comparação indicou que o primeiro valor é menor, salta para a etiqueta "menor".
JBE Salta para uma etiqueta se a comparação anterior for menor ou igual (usando a flag de carry). JBE <etiqueta> JBE menorOuIgual Se a última comparação indicou que o primeiro valor é menor ou igual, salta para a etiqueta "menorOuIgual".
JA Salta para uma etiqueta se a comparação anterior for maior (usando a flag de carry). JA <etiqueta> JA maior Se a última comparação indicou que o primeiro valor é maior, salta para a etiqueta "maior".
JAE Salta para uma etiqueta se a comparação anterior for maior ou igual (usando a flag de carry). JAE <etiqueta> JAE maiorOuIgual Se a última comparação indicou que o primeiro valor é maior ou igual, salta para a etiqueta "maiorOuIgual".
CALL Chama uma sub-rotina e armazena o endereço de retorno na pilha. CALL <etiqueta> CALL subRotina O fluxo de execução será transferido para a linha marcada com a etiqueta "subRotina".
RET Retorna da sub-rotina para o endereço armazenado na pilha. RET RET O fluxo de execução retornará para o endereço armazenado na pilha.
PUSH Coloca um valor na pilha. PUSH <valor> PUSH r0 O valor contido em r0 será colocado na pilha.
POP Remove um valor da pilha e o armazena em um registrador. POP <destino> POP r0 O valor no topo da pilha será removido e armazenado em r0.
DUP Duplica o valor no topo da pilha. DUP DUP O valor no topo da pilha será duplicado.
SWAP Troca os dois valores no topo da pilha. SWAP SWAP Os dois valores no topo da pilha serão trocados.
ROT Rotaciona os três valores no topo da pilha. ROT ROT Os três valores no topo da pilha serão rotacionados.
SETMATSIZE Define o tamanho das matrizes para operações SIMD. SETMATSIZE <tamanho> SETMATSIZE 4 Define o tamanho da matriz para 4 elementos.
VADD Adiciona dois vetores e armazena o resultado em um vetor de destino. VADD <v_dest>, <v_src1>, <v_src2> VADD v0, v1, v2 O vetor v1 será adicionado ao vetor v2 e o resultado será armazenado em v0.
VMUL Multiplica dois vetores e armazena o resultado em um vetor de destino. VMUL <v_dest>, <v_src1>, <v_src2> VMUL v0, v1, v2 O vetor v1 será multiplicado pelo vetor v2 e o resultado será armazenado em v0.
VDIV Divide um vetor pelo outro e armazena o resultado em um vetor de destino. VDIV <v_dest>, <v_src1>, <v_src2> VDIV v0, v1, v2 O vetor v1 será dividido pelo vetor v2 e o resultado será armazenado em v0.
VLOAD Carrega um vetor da memória para um vetor de destino. VLOAD <v_dest>, [<endereço>] VLOAD v0, [10] O vetor armazenado no endereço 10 será carregado em v0.
VSTORE Armazena um vetor de um registrador vetorial em um endereço específico da memória. VSTORE <vsrc>, [<memAddr>] VSTORE v0, [10] O vetor v0 será armazenado a partir do endereço 10 da memória, com cada elemento do vetor sendo salvo em endereços consecutivos.

Exemplos Práticos

Exemplo 1: Movimentação de Dados

MOV r0, 10         ; Move 10 para r0
MOV r1, 20         ; Move 20 para r1
LOAD r2, [100]     ; Carrega o valor do endereço 100 para r2
STORE [200], r1    ; Armazena o valor de r1 no endereço 200
END                ; Mostrar que chegou ao final do programa

Este exemplo mostra como mover valores para registradores, carregar um valor da memória e armazenar um valor na memória.

Exemplo 2: Operações Aritméticas

MOV r0, 5          ; Move 5 para r0
MOV r1, 3          ; Move 3 para r1
ADD r0, r1         ; Adiciona r1 a r0 (r0 = r0 + r1)
SUB r0, r1         ; Subtrai r1 de r0 (r0 = r0 - r1)
MUL r0, r1         ; Multiplica r0 por r1 (r0 = r0 * r1)
DIV r0, r1         ; Divide r0 por r1 (r0 = r0 / r1)
END                ; Mostrar que chegou ao final do programa

Este exemplo ilustra como realizar operações aritméticas básicas: adição, subtração, multiplicação e divisão.

Exemplo 3: Operações Lógicas

MOV r0, 5          ; Move 5 para r0
MOV r1, 3          ; Move 3 para r1
AND r0, r1         ; Realiza AND entre r0 e r1
OR r0, r1          ; Realiza OR entre r0 e r1
XOR r0, r1         ; Realiza XOR entre r0 e r1
NOT r0             ; Inverte os bits de r0
CMP r0, r1         ; Compara r0 e r1 e atualiza as flags
END                ; Mostrar que chegou ao final do programa

Este exemplo demonstra o uso de operações lógicas e comparação de valores.

Exemplo 4: Saltos Condicionais

MOV r0, 5          ; Move 5 para r0
MOV r1, 10         ; Move 10 para r1
CMP r0, r1         ; Compara r0 e r1
JGE maior          ; Salta para 'maior' se r0 >= r1
JLE menor          ; Salta para 'menor' se r0 <= r1
            ; Código para o caso de r0 < r1
menor:
MOV r2, 0      ; Move 0 para r2
JMP fim        ; Salta para o fim
maior:
MOV r2, 1      ; Move 1 para r2
fim:
END                ; Mostrar que chegou ao final do programa

Este exemplo mostra como usar instruções de comparação e salto condicional para alterar o fluxo de execução com base em condições.

Exemplo 5: Manipulação de Pilha

MOV r0, 5          ; Move 5 para r0
PUSH r0            ; Coloca r0 na pilha
DUP                 ; Duplica o valor no topo da pilha (agora temos 5, 5 na pilha)
SWAP                ; Troca os dois valores no topo da pilha (ainda 5, 5)
ROT                 ; Rotaciona os três valores no topo da pilha (agora temos 5, 5, 5)
POP r1             ; Remove o valor do topo da pilha e armazena em r1 (r1 agora é 5)
END                ; Mostrar que chegou ao final do programa

Este exemplo ilustra como usar instruções de pilha para manipular valores, incluindo empilhar, desempilhar, duplicar, trocar e rotacionar valores.

Exemplo 6: Instruções SIMD

MOV [100], 1
MOV [101], 2        ; Inicializa 2 na memória
MOV [102], 3        ; Inicializa 3 na em memória
MOV [103], 4        ; Inicializa 4 na em memória

MOV [104], 2 
MOV [105], 3
MOV [106], 4
MOV [107], 5

; Carrega dois vetores de 4 elementos da memória
VLOAD v0, [100]   ; Carrega primeiro vetor
VLOAD v1, [104]   ; Carrega segundo vetor

; Realiza operações vetoriais
VADD v2, v0, v1   ; Soma os vetores elemento a elemento
VMUL v3, v0, v1   ; Multiplica os vetores elemento a elemento

; Armazena os resultados na memória (corrigindo a ordem dos argumentos)
VSTORE v2, [108]  ; Armazena resultado da soma
VSTORE v3, [112]  ; Armazena resultado da multiplicação
END                ; Mostrar que chegou ao final do programa

Este exemplo demonstra como usar instruções SIMD para operações vetoriais, incluindo carregamento, adição, multiplicação, divisão e armazenamento de vetores na memória.

Exemplo 7: Chamada e Retorno de Sub-rotinas

MOV r0, 5          ; Move 5 para r0
CALL subRotina     ; Chama a sub-rotina 'subRotina'
MOV r1, r0         ; Move o resultado de r0 para r1
JMP fim            ; Salta para o fim
subRotina:
ADD r0, 10     ; Adiciona 10 a r0
RET             ; Retorna da sub-rotina
fim:
END                ; Mostrar que chegou ao final do programa

Este exemplo mostra como chamar uma sub-rotina, executar algumas operações dentro dela e retornar ao ponto de chamada, preservando o estado.

Exemplo 8: Combinando Diversas Instruções

MOV r0, 5          ; Move 5 para r0
MOV r1, 10         ; Move 10 para r1
ADD r0, r1         ; Adiciona r1 a r0 (r0 = 15)
CMP r0, r1         ; Compara r0 e r1
JGE maior          ; Salta para 'maior' se r0 >= r1
MOV r2, 0          ; Se não, move 0 para r2
JMP fim            ; Salta para o fim
maior:
MOV r2, 1          ; Se r0 >= r1, move 1 para r2
fim:
PUSH r2            ; Coloca o valor de r2 na pilha
POP r3             ; Remove o valor do topo da pilha e armazena em r3
END                ; Mostrar que chegou ao final do programa

Este exemplo combina movimentação de dados, operações aritméticas, comparações, saltos condicionais e manipulação de pilha em um único fluxo de execução.

FAQ - Perguntas Frequentes

Como posso saber se um comando é válido?

Verifique a lista de comandos disponíveis na seção de exemplos e na documentação do simulador. Cada comando possui uma descrição e um exemplo de uso.

O que devo fazer se meu código não estiver funcionando?

Primeiro, verifique se há erros de sintaxe ou se você está utilizando comandos inválidos. Utilize a seção de depuração para identificar onde o erro está ocorrendo. Se necessário, consulte a documentação para entender melhor os comandos.

Como posso adicionar breakpoints no meu código?

Clique no número da linha correspondente onde deseja adicionar o breakpoint. O simulador não aceita breakpoints em linhas vazias ou que contenham apenas comentários. Quando um breakpoint é atingido, a execução do código é pausada nessa linha, permitindo que você inspecione o estado do programa. As linhas seguintes não serão executadas até que você continue a execução.

É possível executar código em diferentes larguras de bits?

Sim, você pode selecionar a largura de bits desejada (8, 16 ou 32 bits) na seção de opções do simulador. Isso afetará como os dados são manipulados durante a execução do código.

Onde posso encontrar mais exemplos de código?

Além da seção de exemplos práticos, você pode consultar a documentação do simulador, que contém exemplos adicionais e explicações detalhadas sobre o uso de cada comando.

Contribuição

Se você deseja contribuir com a documentação ou sugerir novos comandos, sinta-se à vontade para entrar em contato através do nosso repositório no GitHub. Agradecemos qualquer feedback ou contribuição!

Como Contribuir

Para contribuir com o projeto, siga estas etapas:

  1. Fork o Repositório: Faça um fork do nosso repositório no GitHub para sua própria conta.
  2. Crie uma Nova Branch: Crie uma nova branch para sua contribuição. Use um nome descritivo que reflita a sua mudança, por exemplo, adicionar-novo-comando.
  3. Faça Suas Alterações: Realize as alterações desejadas na documentação ou no código.
  4. Teste Suas Alterações: Certifique-se de que suas alterações funcionam conforme o esperado e não quebram nada.
  5. Envie um Pull Request: Quando estiver satisfeito com suas alterações, envie um pull request para o repositório original. Descreva claramente as alterações que você fez e o motivo delas.

Diretrizes de Contribuição

Para garantir que sua contribuição seja aceita, siga estas diretrizes:

  • Escreva um código limpo e bem documentado.
  • Adicione testes para novas funcionalidades sempre que possível.
  • Certifique-se de que a documentação esteja atualizada e clara.
  • Respeite as diretrizes de estilo do projeto.
